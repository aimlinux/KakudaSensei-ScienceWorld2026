小中学生向けに、
正しいバイブコーディングを知ってもらうためのアプリとして
ブロックプログラミングのスクラッチで、
バイブコーディングができるようなアプリを作りたいと考えています。
バイブコーディングの、怖い、不便な面もちゃんと知ってもらい、
自分でコードを微調整するということも学んでもらいたいというのがあって、
なので、AIの思考プロセスも、きちんと表示し、
その思考プロセスの吹き出しから、手が出てきて、
ブロックを配置していくみたいなアニメーションで、遊びごごろを出したい。
融通が利かない部分もありつつも、ちゃんと並走して、
与えられたタスクをクリアしていくゲームみたいな感じにしたくて、
具体的には、石を飛び越えて、だけじゃ、ジャンプ力が足りないけど、
石を、120座標で飛び越えて、という風に、具体的に要件定義したり、
実際にでてきたスクラッチの数値を書き換えたりするみたいな風にして
バイブコーディングを体験できるみたいな。
という目的のアプリの
学習データを作成します。テキストファイルで、ルールは

1行1サンプルで input => output か input<TAB>output 形式
例: stone jump => REASON stone ACTION jump VALUE 50

具体的な数値が入っていない場合は、適当な、絶対に失敗する値を入れる
言われていない行動はしない


成功例は以下のように、具体的な数値を入れる（このほかにも、～高さ120で岩を飛び越えて～なども含まれる）

**Stage 1（岩1つ）**
```text
230進んで高さ120でジャンプしてから520進んで => REASON stone ACTION move VALUE 230 ACTION jump VALUE 120 ACTION move VALUE 520
```

**Stage 2（岩2つ）**
```text
130進んで120ジャンプ、50進んで120ジャンプ、最後に370進んで => REASON stone ACTION move VALUE 130 ACTION jump VALUE 120 ACTION move VALUE 150 ACTION jump VALUE 120 ACTION move VALUE 370
```

**Stage 3（大きな穴）**
```text
200進んで150ジャンプしてから450進んで => REASON hole ACTION move VALUE 200 ACTION jump VALUE 150 ACTION move VALUE 450
```
成功例にも失敗例にも、想定されるステージ名を先頭に明記すること（アウトプットには書かない）
